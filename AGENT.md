# AI Agent Instructions - Sorting Visualizer

When working on this project for **eng lolo**, follow these strict instructions:

## Core Principles
1. **No Frameworks**: Stick to vanilla HTML, CSS, and JavaScript.
2. **Minimalist Aesthetic**: 
    - No complex gradients or heavy shadows.
    - Use a "feminine" design language: soft pastel colors (pinks, soft blues, creams), rounded corners (8-12px), and elegant fonts.
    - Focus on clarity and ease of use.
3. **Modularity**:
    - `index.html`: Main structure.
    - `css/style.css`: All styling.
    - `js/visualizer.js`: Handles DOM updates for the sorting bars.
    - `js/sorting-algorithms.js`: Pure algorithm logic with async/await for visualization timing.
    - `js/app.js`: Connects everything.
4. **Documentation & Comments**:
    - Every file must have a header comment explaining its purpose.
    - Complex algorithms must have step-by-step comments.
    - Create a comprehensive `README.md`.
5. **Language**: Always address the user as **eng lolo**.

## Interaction Guide
- **Beginner Friendly**: All code must be easy to understand for a beginner. Avoid "clever" or complex syntax.
- **Variables**: Use `let` and `const` only. Avoid `var`.
- **Minimal Edits**: When changing code, aim for minimal, surgical changes rather than large refactors.
- Provide implementation plans before making large changes.
- Ensure all code is modular and easy for **eng lolo** to learn from.
- Prioritize clean, readable code over overly clever optimizations.
